What are promises and why do we need them?

        Promises are a way to handle asynchronous operations in JavaScript. A promise is an object that represents the state of an operation that may not be completed yet, but will be in the future. Promises allow us to write cleaner and more manageable code than using callbacks or events.

        We need promises because they make it easier to deal with multiple asynchronous operations, avoid callback hell, and handle errors gracefully23. Promises also enable us to use async/await syntax, which makes the code look more synchronous and readable.
        
        Some examples of asynchronous operations that can use promises are:
        
        Fetching data from a server using the fetch() API.
        Reading or writing files using the FileReader or FileWriter API.
        Performing computations in a web worker using the postMessage() and onmessage API.

What is promise chaining

Promise chaining is a technique in JavaScript that allows you to chain multiple asynchronous operations together using promises. The key idea is that a Promise’s then() function returns another promise, so you can chain then() calls together to tell JavaScript to execute async calls in order.

What is the DOM?

The DOM (Document Object Model) is a programming interface for web documents. It represents the page so that programs can change the document structure, style, and content. The DOM represents the document as nodes and objects; that way, programming languages can interact with the page.

What are closures? Give an example of closure

A closure is the combination of a function and the lexical environment (scope) within which that function was declared. Closures are a fundamental and powerful property of JavaScript.

How many operators do we have in JS ?

  • Arithmetic Operators: perform arithmetic on numbers (e.g., +, -, *, /).
  • Assignment Operators: assign values to variables (e.g., =, +=, -=, *=).
  • Comparison Operators: compare two values and return a boolean value (e.g., ==, ===, !=, !==).
  • Logical Operators: used to determine the logic between variables or values (e.g., &&, ||, !).
  • String Operators: used to concatenate strings (e.g., +).
  • Conditional (Ternary) Operator: assigns a value to a variable based on a condition (e.g., ? :).
  • Type Operators: used to determine the data type of a variable (e.g., typeof, instanceof).

What are objects in javascript?

        In JavaScript, an object is a standalone entity with properties and type. Objects can be thought of as a collection of related data in the form of key-value pairs. Here’s an example:

        const car = {
          make: 'Honda',
          model: 'Civic',
          year: 2020
        };
        Copy
        In this example, car is an object with three properties: make, model, and year. Each property has a value associated with it ('Honda', 'Civic', and 2020, respectively).
